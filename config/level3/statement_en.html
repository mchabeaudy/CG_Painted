Goal

Equipped with painting robots you will have to paint the largest area before the time runs out.

Source code is accessible here.

  Rules

During this painting battle you will have to give INSTRUCTION to your robot.
  1. MOVE : Your robot will move
  2. WAIT : Your robot will not do anything

A MOVE instruction must be followed by a DIRECTION .
  1. RIGHT
  2. LEFT
  3. UP
  4. DOWN

When your robot moves it automatically paints the square on which it arrives.
In case of conflict (2 or more robots that want to go on the same square), the robot with the biggest initiative will move on the square while other robots will remain on their space.

Chronology

In this level there is only one step:
Robots MOVE action is performed

The game stands for 100 turns.
Good luck!
 
Victory conditions
  • Your team painted the most square after 100 turns.
 
Lose conditions
  • Your program input not well formatted.
  • Your program input is invalid.
  • Your program doesn't provide an input.

  Game input

Input for a turn

First you receive : 8 integers playerId, team, boardWidth, boardHeight, tpCount, boxCount, robotCount, robotPerPlayer where:

  • playerId : your player id
  • teamId : your team id
  • boardWidth : the board width
  • boardHeight : the board height
  • tpCount : number of teleport portal (0 in this lelvel)
  • boxCount : number of boxes (0 in this level)
  • robotCount : number of all robots
  • robotPerPlayer : number of robot per player (1 in this level)

Then you receive tpCount lines of 3 input not used in this level

Now start the game loop
You receive boardHeight characters lines representing the map.
E for empty square, W for a wall, an integer (team id) for a painted square. In this league there is no wall.

You will receive robotCount lines of 6 integer that describe robots:
  • robotId : robot id
  • robotX : x robot position
  • robotY : y robot position
  • robotOwner : id of robot owner
  • robotTeam : team id of the robot
  • robotInit : robot initiative (used in case of conflict)
  • robotCount : number of all robots
  • robotPerPlayer : number of robot per player (1 in this level)
You will then receive boxCount lines of 2 integers that describe boxes position:
  • boxX : x box position
  • boxY : y box position

playerCount following lines : one line for each player. First line is for player 0, next one for player 1, etc. Each line contains 3 integers startUpId start-up id, marketShare start-up market share in thousandths and reputation start-up reputation

Output for a turn
robotPerPlayer lines (one line in this level) containing robot instruction
